    ## JSX Rules

    <Callout type="info">
      The `jsx-*` rules check for issues exclusive to JSX syntax, which are absent from standard JavaScript (like handwritten `createElement()` calls).
    </Callout>

    **Key Rules:**

    - [`jsx-dollar`](./jsx-dollar) - Prevents unnecessary `$` symbols before JSX expressions
    - [`jsx-key-before-spread`](./jsx-key-before-spread) - Enforces `key` prop placement before spread props
    - [`jsx-no-comment-textnodes`](./jsx-no-comment-textnodes) - Prevents comments from rendering as text
    - [`jsx-no-duplicate-props`](./jsx-no-duplicate-props) - Disallows duplicate props in JSX elements
    - [`jsx-no-iife`](./jsx-no-iife) - Disallows immediately-invoked function expressions in JSX
    - [`jsx-no-undef`](./jsx-no-undef) - Prevents using variables in JSX that are not defined in the scope
    - [`jsx-shorthand-boolean`](./jsx-shorthand-boolean) - Enforces shorthand syntax for boolean props (üîß Fixable, ‚öôÔ∏è Configurable)
    - [`jsx-shorthand-fragment`](./jsx-shorthand-fragment) - Enforces shorthand syntax for fragment elements (üîß Fixable, ‚öôÔ∏è Configurable)
    - [`jsx-uses-react`](./jsx-uses-react) - Marks React variables as used when JSX is present
    - [`jsx-uses-vars`](./jsx-uses-vars) - Marks JSX element variables as used

    ## Component Rules

    <Callout type="info">
      Component rules enforce best practices and prevent common mistakes in `React` component definitions, covering both class and function components.
    </Callout>

    **Key Rule Groups:**

    **Lifecycle & Deprecated APIs:**

    - [`no-component-will-mount`](./no-component-will-mount) - Replaces `componentWillMount` with `UNSAFE_componentWillMount` (üîÑ Codemod, `React` >=16.3.0)
    - [`no-component-will-receive-props`](./no-component-will-receive-props) - Replaces `componentWillReceiveProps` with `UNSAFE_componentWillReceiveProps` (üîÑ Codemod)
    - [`no-component-will-update`](./no-component-will-update) - Replaces `componentWillUpdate` with `UNSAFE_componentWillUpdate` (üîÑ Codemod)
    - [`no-unsafe-component-will-mount`](./no-unsafe-component-will-mount) - Warns about `UNSAFE_componentWillMount` usage
    - [`no-unsafe-component-will-receive-props`](./no-unsafe-component-will-receive-props) - Warns about `UNSAFE_componentWillReceiveProps` usage
    - [`no-unsafe-component-will-update`](./no-unsafe-component-will-update) - Warns about `UNSAFE_componentWillUpdate` usage

    **React 19 Migrations:**

    - [`no-context-provider`](./no-context-provider) - Replaces `<Context.Provider>` with `<Context>` (üîÑ Codemod, `React` >=19.0.0)
    - [`no-forward-ref`](./no-forward-ref) - Replaces `forwardRef` with `ref` as prop (üîÑ Codemod, `React` >=19.0.0)
    - [`no-use-context`](./no-use-context) - Replaces `useContext` with `use` (üîÑ Codemod, `React` >=19.0.0)

    **Component Structure:**

    - [`no-nested-component-definitions`](./no-nested-component-definitions) - Prevents defining components inside other components
    - [`no-nested-lazy-component-declarations`](./no-nested-lazy-component-declarations) - Prevents lazy component declarations inside components
    - [`no-class-component`](./no-class-component) - Disallows class components except error boundaries

    **State Management:**

    - [`no-access-state-in-setstate`](./no-access-state-in-setstate) - Prevents `this.state` access inside `setState` calls
    - [`no-direct-mutation-state`](./no-direct-mutation-state) - Prevents direct `this.state` mutation
    - [`no-set-state-in-component-did-mount`](./no-set-state-in-component-did-mount) - Restricts `setState` in `componentDidMount`
    - [`no-set-state-in-component-did-update`](./no-set-state-in-component-did-update) - Restricts `setState` in `componentDidUpdate`
    - [`no-set-state-in-component-will-update`](./no-set-state-in-component-will-update) - Restricts `setState` in `componentWillUpdate`

    **Props & Keys:**

    - [`no-children-prop`](./no-children-prop) - Disallows passing `children` as prop
    - [`no-missing-key`](./no-missing-key) - Requires `key` prop in list renderings
    - [`no-duplicate-key`](./no-duplicate-key) - Prevents duplicate `key` values
    - [`no-implicit-key`](./no-implicit-key) - Prevents implicit `key` spreading (üß™ Experimental)
    - [`no-unnecessary-key`](./no-unnecessary-key) - Prevents `key` on non-list elements (üß™ Experimental)
    - [`no-array-index-key`](./no-array-index-key) - Warns against using array indices as `keys`

    **Children API:**

    - [`no-children-count`](./no-children-count) - Disallows `Children.count`
    - [`no-children-for-each`](./no-children-for-each) - Disallows `Children.forEach`
    - [`no-children-map`](./no-children-map) - Disallows `Children.map`
    - [`no-children-only`](./no-children-only) - Disallows `Children.only`
    - [`no-children-to-array`](./no-children-to-array) - Disallows `Children.toArray`

    **Performance & Optimization:**

    - [`no-unstable-context-value`](./no-unstable-context-value) - Prevents unstable values in `Context.Provider`
    - [`no-unstable-default-props`](./no-unstable-default-props) - Prevents referential values as default `props`
    - [`no-unnecessary-use-callback`](./no-unnecessary-use-callback) - Warns about unnecessary `useCallback` usage (üß™ Experimental)
    - [`no-unnecessary-use-memo`](./no-unnecessary-use-memo) - Warns about unnecessary `useMemo` usage (üß™ Experimental)
    - [`prefer-use-state-lazy-initialization`](./prefer-use-state-lazy-initialization) - Enforces lazy initialization in `useState`

    ## DOM Rules

    <Callout type="info">
      DOM rules target `React DOM`-specific concerns including security vulnerabilities, deprecated APIs, and DOM property usage.
    </Callout>

    **Security Rules:**

    - [`no-dangerously-set-innerhtml`](./dom-no-dangerously-set-innerhtml) - Disallows `dangerouslySetInnerHTML`
    - [`no-dangerously-set-innerhtml-with-children`](./dom-no-dangerously-set-innerhtml-with-children) - Prevents using `dangerouslySetInnerHTML` with `children`
    - [`no-script-url`](./dom-no-script-url) - Disallows `javascript:` URLs
    - [`no-unsafe-target-blank`](./dom-no-unsafe-target-blank) - Requires `rel="noreferrer noopener"` with `target="_blank"` (üîß Fixable)
    - [`no-missing-iframe-sandbox`](./dom-no-missing-iframe-sandbox) - Enforces `sandbox` attribute on `iframes` (üîß Fixable)
    - [`no-unsafe-iframe-sandbox`](./dom-no-unsafe-iframe-sandbox) - Prevents unsafe `sandbox` combinations

    **Deprecated API Migrations:**

    - [`no-find-dom-node`](./dom-no-find-dom-node) - Disallows `findDOMNode`
    - [`no-flush-sync`](./dom-no-flush-sync) - Disallows `flushSync`
    - [`no-hydrate`](./dom-no-hydrate) - Replaces `ReactDOM.hydrate()` with `hydrateRoot()` (üîÑ Codemod, `React DOM` >=18.0.0)
    - [`no-render`](./dom-no-render) - Replaces `ReactDOM.render()` with `createRoot().render()` (üîÑ Codemod, `React DOM` >=18.0.0)
    - [`no-render-return-value`](./dom-no-render-return-value) - Disallows return value from `ReactDOM.render`
    - [`no-use-form-state`](./dom-no-use-form-state) - Replaces `useFormState` with `useActionState` (üîÑ Codemod, `React DOM` >=19.0.0)

    **DOM Properties:**

    - [`no-missing-button-type`](./dom-no-missing-button-type) - Enforces explicit `type` on `buttons` (üîß Fixable)
    - [`no-namespace`](./dom-no-namespace) - Prevents namespace usage in `React` elements
    - [`no-string-style-prop`](./dom-no-string-style-prop) - Disallows string values for `style` prop
    - [`no-unknown-property`](./dom-no-unknown-property) - Disallows unknown `DOM` properties (üîß Fixable, ‚öôÔ∏è Configurable)
    - [`no-void-elements-with-children`](./dom-no-void-elements-with-children) - Prevents `children` in void elements

    ## Web API Rules

    <Callout type="info">
      Web API rules prevent resource leaks by detecting uncleaned Web API subscriptions in components and hooks.
    </Callout>

    Implemented Rules:

    - [`no-leaked-event-listener`](./web-api-no-leaked-event-listener) - Prevents leaked `addEventListener` calls
    - [`no-leaked-interval`](./web-api-no-leaked-interval) - Prevents leaked `setInterval` calls
    - [`no-leaked-resize-observer`](./web-api-no-leaked-resize-observer) - Prevents leaked `ResizeObserver` instances
    - [`no-leaked-timeout`](./web-api-no-leaked-timeout) - Prevents leaked `setTimeout` calls

    <Callout type="info">
      Extended hooks rules that supplement the official `eslint-plugin-react-hooks` with additional patterns.
    </Callout>

    - [`no-direct-set-state-in-use-effect`](./hooks-extra-no-direct-set-state-in-use-effect) - Disallows direct `setState` calls in `useEffect` (üß™ Experimental)

    ## Naming Convention Rules

    <Callout type="info">
      Naming convention rules enforce consistent naming patterns for `React` entities.
    </Callout>

    - [`component-name`](./naming-convention-component-name) - Enforces naming conventions for components (‚öôÔ∏è Configurable)
    - [`context-name`](./naming-convention-context-name) - Enforces context names ending with `Context` suffix
    - [`filename`](./naming-convention-filename) - Enforces consistent file naming (‚öôÔ∏è Configurable)
    - [`filename-extension`](./naming-convention-filename-extension) - Enforces JSX file extensions (‚öôÔ∏è Configurable)
    - [`use-state`](./naming-convention-use-state) - Enforces symmetric naming of `useState` pairs (e.g., `[value, setValue]`)

    ## Debug Rules

    <Callout type="warning">
      Debug rules report `React` patterns for code metrics, transformations, or custom tooling. These are not included in the unified plugin by default.
    </Callout>

    - [`class-component`](./debug-class-component) - Reports all class components in JSON format
    - [`function-component`](./debug-function-component) - Reports all function components in JSON format
    - [`hook`](./debug-hook) - Reports all React Hooks in JSON format
    - [`is-from-react`](./debug-is-from-react) - Reports all identifiers initialized from React in JSON format
    - [`is-from-ref`](./debug-is-from-ref) - Reports all identifiers initialized or derived from refs in JSON format
    - [`jsx`](./debug-jsx) - Reports all JSX elements and fragments in JSON format
