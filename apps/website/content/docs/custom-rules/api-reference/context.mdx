The `ctx` object is the first argument passed to `make`. It is the standard ESLint [`RuleContext`](https://eslint.org/docs/latest/extend/custom-rules#the-context-object) from `@typescript-eslint/utils`, with an enhanced `report` method that accepts a `message` string directly instead of requiring a `messageId`.

```ts
{
  name: "my-rule",
  make: (ctx, kit) => {
    // ctx is available here
    return { /* visitor */ };
  },
}
```

## `ctx.report(descriptor)` — Report a Problem

Reports a lint violation at the given AST node. This is the primary way to surface errors and warnings to the user.

The `descriptor` object accepts the following properties:

| Property    | Type                      | Required | Description                                                            |
| :---------- | :------------------------ | :------- | :--------------------------------------------------------------------- |
| `node`      | `TSESTree.Node`           | Yes      | The AST node to report the problem on.                                 |
| `message`   | `string`                  | Yes      | The error message shown to the user.                                   |
| `data`      | `Record<string, unknown>` | No       | Placeholder data interpolated into the message.                        |
| `loc`       | `SourceLocation \| Position` | No    | Override the reported location (instead of the full node range).       |
| `fix`       | `(fixer: RuleFixer) => RuleFix \| RuleFix[] \| null` | No | An auto-fix function. |
| `suggest`   | `Suggestion[]`            | No       | An array of suggested fixes the user can apply manually.               |

### Basic report

```ts
ctx.report({
  node,
  message: "Function components must be defined with arrow functions.",
});
```

### Report with data interpolation

```ts
ctx.report({
  node,
  message: `Custom hook "${hook.name}" doesn't call any hooks.`,
});
```

### Report with an auto-fix

The `fix` function receives a `RuleFixer` and must return a `RuleFix`, an array of `RuleFix`, an iterator of `RuleFix`, or `null` to skip the fix:

```ts
ctx.report({
  node,
  message: "Use arrow function instead.",
  fix(fixer) {
    return fixer.replaceText(node, replacementText);
  },
});
```

### Report with suggestions

Suggestions are user-selectable fixes shown in the editor. Each suggestion needs a `desc` and a `fix`:

```ts
ctx.report({
  node,
  message: "Function components must be defined with arrow functions.",
  suggest: [
    {
      desc: "Convert to arrow function.",
      fix(fixer) {
        return fixer.replaceText(node, replacementText);
      },
    },
  ],
});
```

## `ctx.sourceCode` — Source Code Access

The `sourceCode` property provides read access to the source text and AST. It is the most commonly used property alongside `report`.

### `ctx.sourceCode.getText(node?)`

Returns the source text for a given AST node. When called without arguments, returns the entire file text.

```ts
const src = ctx.sourceCode;
const paramText = src.getText(node.params[0]);
const bodyText = src.getText(node.body);
```

### `ctx.sourceCode.text`

The raw source text of the entire file as a string. Useful for fast-path checks before running expensive analysis:

```ts
// Skip files that don't contain the relevant code
if (!ctx.sourceCode.text.includes("dangerouslySetInnerHTML")) {
  return {};
}
```

### `ctx.sourceCode.ast`

The root `Program` node of the AST. Useful when you need to insert text at the top of the file:

```ts
fix(fixer) {
  return fixer.insertTextBefore(
    ctx.sourceCode.ast,
    'import { createRoot } from "react-dom/client";\n',
  );
}
```

### `ctx.sourceCode.getScope(node)`

Returns the scope for the given AST node, as defined by `@typescript-eslint/scope-manager`:

```ts
const scope = ctx.sourceCode.getScope(node);
```

## `RuleFixer` Methods

The `fixer` object passed to `fix` and `suggest` callbacks provides these methods:

| Method                              | Description                                      |
| :---------------------------------- | :----------------------------------------------- |
| `fixer.replaceText(node, text)`     | Replace the text of a node.                      |
| `fixer.replaceTextRange(range, text)` | Replace text in a character range.             |
| `fixer.insertTextBefore(node, text)` | Insert text before a node.                      |
| `fixer.insertTextAfter(node, text)`  | Insert text after a node.                       |
| `fixer.insertTextBeforeRange(range, text)` | Insert text before a character range.     |
| `fixer.insertTextAfterRange(range, text)`  | Insert text after a character range.      |
| `fixer.remove(node)`               | Remove a node.                                   |
| `fixer.removeRange(range)`          | Remove a character range.                        |

A `fix` function can return a single fix, an array of fixes, or `null` to skip:

```ts
fix(fixer) {
  return [
    fixer.insertTextBefore(ctx.sourceCode.ast, 'import { hydrateRoot } from "react-dom/client";\n'),
    fixer.replaceText(node, `hydrateRoot(${containerText}, ${elementText})`),
  ];
}
```

## Full Example

Putting it all together — a rule that enforces arrow function components with a suggested fix:

```ts
import { defineConfig as defineReactConfig, merge } from "@eslint-react/kit";

defineReactConfig({
  name: "function-component-definition",
  make: (ctx, kit) => {
    const { query, visitor } = kit.collect.components(ctx);

    return merge(visitor, {
      "Program:exit"(program) {
        for (const { node } of query.all(program)) {
          if (node.type === "ArrowFunctionExpression") continue;
          ctx.report({
            node,
            message: "Function components must be defined with arrow functions.",
            suggest: [
              {
                desc: "Convert to arrow function.",
                fix(fixer) {
                  const src = ctx.sourceCode;
                  const prefix = node.async ? "async " : "";
                  const params = `(${node.params.map((p) => src.getText(p)).join(", ")})`;
                  const returnType = node.returnType ? src.getText(node.returnType) : "";
                  const body = src.getText(node.body);

                  if (node.type === "FunctionDeclaration" && node.id) {
                    return fixer.replaceText(
                      node,
                      `const ${node.id.name} = ${prefix}${params}${returnType} => ${body};`,
                    );
                  }

                  return null;
                },
              },
            ],
          });
        }
      },
    });
  },
});
```
