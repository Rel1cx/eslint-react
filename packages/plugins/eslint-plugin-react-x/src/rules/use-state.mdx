## Description

Enforces destructuring and symmetric naming of the `useState` hook value and setter.

This rule ensures two things:

1. The `useState` hook is destructured into a value and setter pair (opt-in via `enforceAssignment`).
2. The setter is named symmetrically to the value — `set` followed by the capitalized value name (e.g. `count` → `setCount`). The state value must be a plain identifier; destructured patterns (`ObjectPattern`, `ArrayPattern`, etc.) are not supported and are reported as an invalid assignment form.

## Options

```ts
type Options = {
  enforceAssignment?: boolean; // default: true
  enforceSetterName?: boolean; // default: true
};
```

### `enforceAssignment`

When `true`, requires `useState` to be destructured into a `[value, setter]` array pattern. Bare assignments (e.g. `const state = useState(0)`) or calls without any assignment are reported.

Default: `true`

### `enforceSetterName`

When `true`, requires the setter to be named `set` + capitalized value name (e.g. `setCount` for `count`). Snake-case and object-pattern state names are normalized before comparison.

Default: `true`

## Examples

### Failing

```tsx
// ❌ Setter name does not match value name
import { useState } from "react";

function Counter() {
  const [count, updateCount] = useState(0);
  //            ^^^^^^^^^^^
  //            - The setter should be named 'set' followed by the capitalized state
  //              variable name, e.g. 'setCount' for 'count'.
  return <div>{count}</div>;
}
```

```tsx
// ❌ Setter prefix is correct but capitalization is wrong
import { useState } from "react";

function Counter() {
  const [count, setcount] = useState(0);
  return <div>{count}</div>;
}
```

```tsx
// ❌ Not destructured (enforceAssignment: true)
import { useState } from "react";

function Counter() {
  const count = useState(0);
  //    ^^^^^
  //    - useState should be destructured into a value and setter pair,
  //      e.g. const [state, setState] = useState(...).
  return <div>{count}</div>;
}
```

### Notes

- When the state value is a destructured pattern (e.g. `ObjectPattern` or `ArrayPattern`), the rule reports `invalidAssignment` — there is no unambiguous canonical name to derive a setter from, so the destructuring form itself is considered invalid.


### Passing

```tsx
// ✅ Symmetric camelCase naming
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);
  return <div>{count}</div>;
}
```

```tsx
// ✅ Snake_case value and setter
import { useState } from "react";

function Counter() {
  const [foo_bar, set_foo_bar] = useState(0);
  return <div>{foo_bar}</div>;
}
```

```tsx
// ❌ Object-pattern value — use a plain identifier instead
import { useState } from "react";

function Form() {
  const [{ foo, bar }, setFooBar] = useState({ foo: "a", bar: "b" });
  //     ^^^^^^^^^^^
  //     - useState should be destructured into a value and setter pair,
  //       e.g. const [state, setState] = useState(...).
  return <div>{foo}</div>;
}
```

```tsx
// ✅ Object state stored under a plain identifier
import { useState } from "react";

function Form() {
  const [form, setForm] = useState({ foo: "a", bar: "b" });
  return <div>{form.foo}</div>;
}
```

```tsx
// ✅ Only value destructured — setter omitted (enforceSetterName skipped)
import { useState } from "react";

function Component() {
  const [value] = useState(() => expensiveSetup());
  return <div>{value}</div>;
}
```
